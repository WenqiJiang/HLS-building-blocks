# Priority Queue

Conlusion: Use priority_queue_optimized if possible, because in HLS it's hard to gain (a) high performance / low resource consumption and (b) flexibility in the mean time. 

Resource Consumption: LUT(priority_queue_flexible) ~= 2 x LUT(priority_queue_optimized)

Performance: 2 CC / insertion. priority_queue_optimized has lower head / tail overhead than priority_queue_flexible.

## priority_queue_optimized

Optimized for resource consumption and performance. Everything is wrapped in a single function.

The wrapper function: take x inputs from input stream to the priority queue, and then return the contents in the queue to output stream.
